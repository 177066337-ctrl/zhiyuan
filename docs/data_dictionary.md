# 数据字典

## 一、数据来源

本阶段只处理以下 3 个 Excel 文件：

1. `全国普通高等学校名单.xls`
2. `普通高等学校本科专业目录.xlsx`
3. `普通高等学校高等职业教育（专科）专业目录.xlsx`

未处理内容：

- PDF
- Word
- PPT
- 推荐算法
- 网页开发

## 二、schools.json 字段字典

| 字段名 | 类型 | 含义 | 来源 | 清洗说明 |
|---|---|---|---|---|
| `school_id` | string | 稳定院校 ID | 由 `school_name + province + city` 生成 | 使用确定性哈希生成，不使用随机数 |
| `school_name` | string | 院校名称 | `学校名称` | 去除前后空格和换行 |
| `province` | string | 省份 | 省份分组标题行，如 `北京市（91所）` | 由分组标题解析并回填到后续院校记录 |
| `city` | string | 城市/所在地 | `所在地` | 原表字段名为所在地，第一版直接保留其文本，不强行拆分市级 |
| `department` | string | 主管部门 | `主管部门` | 去除前后空格 |
| `school_level` | string | 办学层次 | `办学层次` | 统一归并为 `本科`、`专科` 或原文本 |
| `school_type` | string | 院校类型 | 无直接来源 | 第一版留空 |
| `ownership` | string | 办学性质 | `主管部门`、`备注`、`学校名称` | 仅做保守推断：`中外合作`、`内地与港澳台合作`、`民办`、`公办`、`其他` |
| `tags` | array | 标签列表 | 无直接来源 | 第一版为空数组 |
| `official_site` | string | 官网 | 无直接来源 | 第一版为空字符串 |
| `source_file` | string | 来源文件名 | 文件元信息 | 固定写入源文件名 |
| `source_row` | number | 来源行号 | Excel 行号 | 便于回溯原始数据 |

## 三、majors.json 字段字典

| 字段名 | 类型 | 含义 | 来源 | 清洗说明 |
|---|---|---|---|---|
| `major_id` | string | 稳定专业 ID | 由 `degree_level + major_code + major_name` 生成 | 使用确定性哈希生成，不使用随机数 |
| `major_code` | string | 专业代码 | `专业代码` | 去除空格，本科代码统一转大写 |
| `major_name` | string | 专业名称 | `专业名称` | 去除前后空格和换行 |
| `major_category` | string | 专业类 | 本科目录中的 4 位代码层级；专科目录中的专业类 | 由层级标题回填 |
| `major_discipline` | string | 门类/大类 | 本科目录中的 `学科门类`；专科目录中的 `专业大类` | 由层级标题回填 |
| `degree_level` | string | 培养层次 | 来源文件语义 | 本科目录写 `本科`，专科目录写 `专科` |
| `degree` | string | 授予学位 | 无直接来源 | 第一版留空 |
| `duration` | string | 学制 | 无直接来源 | 第一版留空 |
| `subject_requirement` | string | 选科要求 | 无直接来源 | 第一版留空 |
| `description` | string | 专业简介 | 无直接来源 | 第一版留空 |
| `source_file` | string | 来源文件名 | 文件元信息 | 固定写入源文件名 |
| `source_row` | number | 来源行号 | Excel 行号 | 便于回溯原始数据 |

## 四、字段来源说明

### 1. `全国普通高等学校名单.xls`

主表头为：

- `序号`
- `学校名称`
- `主管部门`
- `所在地`
- `办学层次`
- `备注`

其中省份并不是单独列，而是通过类似 `北京市（91所）` 的分组标题行表达。转换时把该标题识别为当前省份，并回填给之后的院校记录。

### 2. `普通高等学校本科专业目录.xlsx`

主表头为：

- `专业代码`
- `专业名称`

该表是层级目录，不是平铺明细表。层级规则如下：

- 2 位代码：学科门类
- 4 位代码：专业类
- 6 位或带字母后缀代码：具体专业

### 3. `普通高等学校高等职业教育（专科）专业目录.xlsx`

主目录 sheet 表头为：

- `序号`
- `专业代码`
- `专业名称`
- `原专业代码`
- `原专业名称`
- `调整情况`

增补专业 sheet 表头为：

- `序号`
- `专业大类`
- `专业类`
- `专业代码`
- `专业名称`

## 五、字段清洗规则

### 通用规则

1. 去除字段值前后空格。
2. 去除换行，并把连续空白压缩成单个空格。
3. 过滤空行。
4. 过滤标题说明行。
5. 保留 `source_file` 和 `source_row`，确保可追溯。

### schools 规则

1. 先识别省份分组行，再处理院校数据行。
2. `school_level` 按原表 `办学层次` 归并到 `本科` / `专科` / 其他。
3. `ownership` 只做保守推断，不做过度猜测：
   - 含 `中外合作` -> `中外合作`
   - 含 `港澳` / `港澳台` -> `内地与港澳台合作`
   - 含 `民办` -> `民办`
   - `主管部门` 明显属于政府、教育行政部门等 -> `公办`
   - 其他情况 -> `其他`

### majors 规则

1. 本科目录只输出具体专业行，不输出门类行和专业类行。
2. 专科目录只输出正式专业记录，不输出目录行和续行。
3. 对同一来源下重复出现的专科增补专业，按 `major_id` 去重。

## 六、待确认字段

以下字段在当前 3 个 Excel 中没有可靠来源，或含义不能直接确认，因此没有强行补齐：

- `school_type`
- `official_site`
- `tags`
- `degree`
- `duration`
- `subject_requirement`
- `description`

另外，以下判断仅是保守推断，后续最好补充独立数据源复核：

- `ownership`
- `city` 是否应进一步拆分成标准城市名
- `school_level` 中非标准文本是否需要再归一化

## 七、暂时缺失字段说明

当前数据暂时缺失，且会直接影响后续筛选和展示体验的字段包括：

- 院校类型
- `985/211/双一流` 标签
- 官网
- 院校简介
- 专业简介
- 选科要求
- 学制
- 授予学位

结论：

- 当前数据足够支撑第一版院校查询和专业查询的基础检索。
- 如果希望做更有价值的筛选体验，下一步应优先补充院校标签数据，而不是直接进入复杂推荐逻辑。
