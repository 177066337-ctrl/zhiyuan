# Project Working Rules

## Data Safety

- Do not delete raw source materials under the project root.
- Do not overwrite user-provided source files in place.
- Keep original filenames and preserve raw files as read-only references.

## Directory Rules

- Put all data conversion and validation scripts in `scripts/`.
- Put all web application code in `app/`.
- Put all project documentation in `docs/`.
- Put normalized static JSON output in `app/public/data/`.

## Delivery Rules

- Complete work phase by phase and summarize at the end of each phase.
- When a field meaning is unclear, record the uncertainty in documentation instead of inventing facts.
- Keep the recommendation algorithm transparent, parameterized, and explainable.
- Do not use absolute admissions promises or guaranteed-offer wording.

## Current Scope

- Current task is limited to workspace audit and documentation.
- Do not start web development before data inventory and cleaning plan are confirmed.
